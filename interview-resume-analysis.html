核心技术能力（你可能不熟悉的解释）

  • Hive SQL / Data Lake：和普通 SQL 一样写查询，但数据量可能是几十亿行，跑在分布式集群上。ByteDance 内部叫"数仓"，是公司级数据中台的核心。
  • Power BI：微软的商业智能报表工具，拖拖拽拽就能做出带图表的数据看板，不用写前端代码，适合给非技术业务方汇报数据。
  • A/B 测试：把用户分成两组，一组看旧版、一组看新版，统计哪个版本的购买率/点击率更高。TikTok 做的是亿级用户的 A/B，结论非常可靠。
  • LLM 代码生成评测：让 AI（GPT-4o、DeepSeek 等）自动写 Python/SQL 代码，然后用数据验证代码质量——正确率、效率、风格——并交叉对比不同模型。
  • Prompt Engineering：通过精心设计发给 AI 的指令（prompt），让 AI 输出更准确的结果。她用这个提升了数据清洗的质量。
  • ETL：Extract（抽数据）→ Transform（清洗/转换）→ Load（写入目标库）。数据管道的标准三步，是数据分析师日常工作的基础。
  • SSRS / InfoMaker：企业级报表工具，主要用于财务/运营等内部定时报表，技术含量中等，但说明她做过传统企业 IT 环境。

共 5 类题型。建议面试顺序：破冰 → 技术 → 行为 → AI → 业务 → 文化匹配。 点击展开看参考答案和追问方向。

🔧 技术能力类（验证真实深度）

技术 你在 TikTok 搭建电商搜索指标体系时，从零开始最难的一步是什么？你是怎么选定核心指标的？
参考答案要点

好的候选人会提到：从业务目标（GMV? 用户体验？）反推，定义北极星指标，再拆解为过程指标（搜索渗透率、点击率、转化率）。难点往往是「指标口径对齐」——同一个"搜索量"不同部门定义可能不同，需要拉齐。

如果她能说出「我用什么方法判断一个指标是否够好」（比如：可监控、可拆解、能驱动决策），加分。

追问：「如果指标之间出现矛盾（搜索量上升但转化率下降），你怎么处理？」

技术 描述你在 ByteDance 做过的一个 A/B 测试的完整流程，从假设到结论。
参考答案要点

她有商品卡展示逻辑优化的具体案例，可以让她展开说。关注点：

  • 实验设计：如何分流（随机/分层）？样本量怎么算？实验周期多长？
  • 指标选择：主指标 vs 护栏指标（防止为了一个指标牺牲另一个）
  • 结论判断：用统计显著性（p-value）还是贝叶斯方法？
  • 上线后：有没有持续监控？有没有出现反弹？

追问：「实验期间发现数据异常怎么处理？」

技术 你最常用 SQL 处理哪类问题？能写一段处理漏斗分析的 SQL 思路吗？
参考答案要点

漏斗分析（比如：搜索 → 点击 → 加购 → 下单）标准写法是用 SUM(CASE WHEN step = 'X' THEN 1 ELSE 0 END) 或窗口函数。

好的候选人会提到：如何处理用户重复行为（去重）？如何定义时间窗口（7天内完成转化算不算）？

如果她用过 WITH CTE / 窗口函数 / Hive 的 LATERAL VIEW，说明 SQL 深度够。

追问：「同一个 SQL 查询，用户量从 100 万增到 10 亿，你怎么优化？」

🤖 AI / Prompt 工程类（核心差异点）

AI 你在评测 LLM 代码生成质量时，具体用了哪些评估标准？怎么判断 GPT-4o 比 DeepSeek 好还是差？
参考答案要点

这是她在 ByteDance AIGC 团队的核心工作，应该很熟。期望听到：

  • 自动化指标：代码执行通过率（Pass@k）、语法正确率、效率（执行时间）
  • 人工评估：可读性、风格一致性、是否符合业务规范（比如命名规范）
  • 业务指标：使用这些代码后，业务效率有没有提升？开发者采用率？

如果她说「我们有 gold standard test set」或「用 human annotation 做评测」，加分。

追问：「如果两个模型在不同场景下各有优劣，你怎么给业务方一个最终建议？」

AI 你用 Prompt Engineering 提升数据质量，具体是什么场景？Prompt 怎么写的？
参考答案要点

关注她的迭代方法论：她是怎么判断一个 prompt 写得好不好的？有没有 A/B 测试 prompt？有没有版本管理 prompt？

好的答案示例：「我用 few-shot 示例让模型输出标准化格式，用 chain-of-thought 让模型先分析数据特征再决定是否异常，然后用 20 个标注样本验证准确率从 72% 到 89%。」

追问：「如果同一个 prompt 在不同数据集上效果差异很大，你怎么分析根因？」

AI 【场景题】我们有个 AI 自动报价功能，给出报价后 30% 的情况客户说"价格不合理"。你会设计什么分析方案来定位问题？
参考答案框架（你可以用这个评分）
  • 第一步：数据收集 — 那 30% 的案例有什么共同特征？（产品类别、客户区域、历史成交价差距？）
  • 第二步：根因分类 — 是 AI 输入数据有问题（市场价参考不准）？还是 AI 推理逻辑有问题？还是客户预期管理问题？
  • 第三步：实验方案 — 针对每个假设，设计最小验证实验
  • 第四步：指标监控 — 修复后，定义「成功」的量化标准

如果她主动问「30% 是比历史高还是低？」「不合理是指价格太高还是太低？」——加分，说明有业务意识。

💬 行为 / STAR 类（判断做事方式）

行为 说说你在 ByteDance 做过最有成就感的数据项目，以及你在里面的具体贡献。
评估维度
  • 主动性：是她发现问题发起的，还是领导安排的？
  • 端到端：从问题发现到落地有没有全程参与，还是只做了某一段？
  • 量化结果：她的工作对最终 GMV / 指标有多大贡献？她能说清楚数字吗？
  • 跨部门协作：数据分析师通常需要推动工程/算法/产品改变，她怎么做的？

追问：「如果重来一次，你会做什么不一样的选择？」

行为 你负责的 AIGC 团队只有 6 个人。作为数据分析师，你和工程师/算法工程师的协作边界是什么？
参考答案要点

这个问题考察她的角色认知：她是"给工程师喂需求"，还是"和工程师平等设计实验"？

在早期公司，数据分析师往往需要自己写脚本处理数据，不能总等工程师。她有没有这个意识？

同时观察：她在大公司有没有"等资源"的惯性？能不能适应小团队自驱的工作方式？

行为 给我讲一个你的数据分析结论最终和业务方直觉相反、你是怎么处理的。
参考答案要点

好的候选人既不会「数据永远对」，也不会「领导永远对」。期望看到：

  • 先理解业务方的直觉来源（历史经验？还是某个特例？）
  • 检查自己的数据假设是否完整（有没有遗漏某个变量？）
  • 用数据重新讲故事，找到双方都能接受的框架
  • 最终结果：谁的判断更接近事实？她怎么复盘？

📊 业务理解类（考察能否快速适配）

业务 你在 TikTok 做高价值用户的搜索优化。如果换成 B2B 工业品贸易，"高价值客户"怎么定义？你会用哪些维度？
评估思路

这是一个类比迁移的题。B2B 高价值客户通常是：订单金额大、复购频率高、付款周期短、产品品类集中（采购标准化）、决策周期短。

如果她能快速提出「我会先看历史成单金额、然后看采购频率、再看 BD 跟进深度（这个正好是 make-log 记录的）」——说明她迁移能力强。

追问：「如果 BD 销售的 CRM 数据不完整，你会怎么补全这个分析？」

业务 我们的 BD 团队每天在 Slack 上记录客户拜访日志。你觉得从这些文本数据里，能挖掘什么有价值的分析？
参考答案框架
  • 活动分析：每个 BD 的拜访量、拜访时间分布、拜访客户类型占比
  • 效能分析：拜访次数 → 报价次数 → 成单次数的漏斗转化率，找出瓶颈
  • 内容分析：客户拒绝的高频原因是什么？成单客户有什么共同特征？
  • 预测分析：根据历史数据，哪类拜访记录最可能在 30 天内成单？

如果她主动提到「用 NLP 对日志文本做分类」或「用机器学习预测成单概率」——加分。

🎯 文化匹配 / 动机类

匹配 你在 ByteDance 是一个成熟大团队，这里是早期创业公司。你期待这两者有什么不同？最担心什么？
评估维度

听她怎么描述「担心」——如果她担心的是「基础设施不完善、数据工具链原始」，这是很真实的，说明她想清楚了。

警惕信号：如果她说「没什么担心的」（可能没想清楚）或者「担心工资少」（重点放在待遇而非成长）。

理想信号：「我希望能更直接看到自己分析的业务影响，大公司做一个项目影响可能要等 6 个月，这里也许 2 周就能看到。」

匹配 如果给你 3 个月，你觉得你能为我们的产品做出什么具体的贡献？
评估维度

这是反向测试：她有没有做过功课了解我们的产品？她给出的答案是通用的（「我会建立指标体系」）还是具体的？

理想回答：「如果你们的 BD 日志已经积累了半年数据，我可以先做一个 BD 效能报告，找出成单率最高的拜访模式；同时评估 AI 报价的准确率，给出改进建议。」
